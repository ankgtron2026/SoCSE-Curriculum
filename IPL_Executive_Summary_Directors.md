# Intense Programming Labs (IPL): Proposal for Four Core Courses
### School Directors Brief | For Implementation Review

---

## Problem

Our placement-critical programming courses—**Introduction to C Programming (ICP), Data Structures (DS), Database Management Systems (DBMS), and Object-Oriented Programming (OOP)**—face a critical challenge: current dual-level assessment within a single course cannot provide the depth and hands-on project experience that placement-ready students need. Motivated students lack a structured pathway for advanced inquiry, and their portfolio value remains underdeveloped. The awareness floor is protected, but the advanced ceiling remains constrained.

---

## Proposed Solution: Intense Programming Labs (IPL)

Launch an **optional, facilitated advanced learning stream** (IPL) parallel to each core course:

- Both streams take the **same assessment** (identical exam, same grading scale) — preserving grade fairness and CGPA coherence.
- The IPL cohort engages in **project-driven, peer-led inquiry** with faculty facilitation, not additional coursework.
- The awareness stream maintains structured, exam-focused core delivery.
- IPL produces **portfolio artifacts and institutional badges** — placement signals that add value *beyond* grades.

**Key principle:** *IPL deepens practice and portfolio-building, not assessment burden.*

---

## Student Selection

- **Eligibility:** Voluntary opt-in; no prerequisites other than course enrollment.
- **Target:** Motivated students (prior SIA ≥ 7.0 or faculty recommendation).
- **Cohort size:** 15–25 students per course to ensure facilitation quality.
- **Diversity outreach:** Faculty actively invite underrepresented learners to prevent self-selection bias.
- **Governance:** Program leads approve final enrollment lists before semester start.

---

## Pedagogy & Instruction in IPL

**Structure:**
- **Weekly project cycles** aligned to core exam topics (e.g., DS: heap/graph data structures; DBMS: indexing; OOP: design patterns).
- **Facilitator-led guidance**, not lectures—students work in small teams on real-world design challenges.
- **Peer debugging and code reviews** to develop systems thinking and professional practices.
- **Reflection journals** to connect project work back to theory and exam-aligned concepts.

**Mapping to courses:**
- **ICP:** System-level programs (process management, file I/O, memory safety).
- **DS:** Production-style implementations (priority queues, memory-efficient graphs, search heuristics).
- **DBMS:** Query engine components, transaction logging, normalization design.
- **OOP:** Architected domain models, design-pattern-driven systems, testable APIs.

---

## Assessment

- **Same exam as awareness stream:** Identical midterm and final exam; same grading rubric.
- **Portfolio grading (internal, non-exam):**
  - Technical correctness and system design quality.
  - Code documentation and architecture clarity.
  - Iteration evidence and debugging reflection.
  - Rubric: exemplary/proficient/developing.
- **No grade premium for IPL work**—instead, portfolio quality feeds into badge criteria and placement signals.
- **Outcome:** CGPA remains comparable across streams; advanced depth is captured in non-grade signals.

---

## Motivation: Portfolio + Badges

**Portfolio:**
- Each student maintains a **GitHub-based portfolio** with 2–3 projects per semester.
- Portfolio includes: problem statement, architecture notes, testing evidence, learning reflection.
- Each project explicitly maps to course competencies and placement-relevant skills.
- Portfolio is recruiter-ready—can be shared during placement interviews.

**Institutional Badge:**
- **"Depth Certified"** badge issued upon:
  1. Passing the core course exam (same as awareness stream).
  2. Completing IPL portfolio projects to proficient standard.
  3. Demonstrating engagement in inquiry-based learning.
- Badge appears on placement portal, transcript supplement, and recruiter-facing credentials.
- **Placement value:** Priority consideration for internships and domain-aligned roles.

**Recruiter alignment:**
- Placement office briefs recruiters: "Badge earners mastered the core and completed advanced practice."
- Students can demo portfolio projects during campus recruitment.
- Badge acts as a **placement multiplier**, not a grade substitute.

## Success Metrics

- Awareness stream exam performance unchanged or improved.
- IPL opt-in rate ≥ 20% across four courses.
- Portfolio artifacts rated proficient or exemplary in ≥ 80% of submissions.
- At least one external employer recognizes the badge in placement conversations.
- Faculty report operational feasibility and satisfaction with facilitation model.

---

**Recommendation:** Approve IPL pilot for the four core programming courses to deepen student expertise, protect placement readiness, and signal institutional commitment to advanced learning without grade distortion.
